# 📋 BiasLens PRD (Revised · 기준 기반 버전)

---

## 1. 문제 정의 (Problem Definition)

### 배경

- 디지털 뉴스 환경에서 사용자는 하루 수십 개의 기사를 접하지만
    
    **뉴스를 해석하고 판단할 ‘공통 기준’은 점점 사라지고 있음**
    
- 한국언론진흥재단(2024)에 따르면
    
    **뉴스 소비자의 72.1%가 뉴스를 의도적으로 회피**
    
- 뉴스 회피의 주요 이유:
    - “사실과 의견이 섞여 있어서”
    - “표현이 감정적·단정적으로 느껴져서”
- 그러나 현재 뉴스 서비스들은:
    - 조회수·체류시간 중심 추천
    - 자극적 제목 강조
    - 개별 기사 소비에 최적화
        
        → **기사 ‘표현 방식’을 비교·설명해주는 도구는 부재**
        

---

### 핵심 문제

- 뉴스 소비자는 다음 질문에 스스로 답하기 어렵다:
    - 같은 이슈를 언론마다 **어떻게 다르게 표현하고 있는가?**
    - 특정 매체의 표현 방식은 **일관된 경향을 보이는가?**
    - 내가 중요하게 여기는 기준에서 **이 표현은 얼마나 과한가?**
- 그 결과:
    - 판단 기준이 개인 인상·이미지·SNS 담론에 의존
    - “느낌상 그렇다”는 불신만 누적
- **사용자 기준에 따라 비교 가능한 표현 지표의 부재**가 문제의 핵심

---

### 문제 정의 한 문장 (Revised)

> 뉴스 소비자는 기사 표현 방식의 차이를 비교할 수 있는 기준과 지표 없이,
> 
> 
> 개인적 인상에 의존해 뉴스를 해석하고 있으며,
> 
> 이로 인해 뉴스 신뢰 저하와 회피가 반복되고 있다.
> 

---

## 2. 목표 (Goals)

### Primary Goals

1. 뉴스 기사 텍스트에서
    
    **표현 패턴(가치 판단, 단정성, 표현 강도 등)을 구조적으로 추출**
    
2. 사용자가 선택한 기준에 따라
    
    **기사·매체 단위 표현 차이를 수치화**
    
3. AI가 판단을 대신하지 않고
    
    **사용자가 스스로 비교·해석할 수 있는 근거 제공**
    
4. 개인정보·저작권 리스크 없이
    
    **공개 기사 기반 분석 구조 유지**
    

---

### Business / Product Goals

- 뉴스 소비를 위한 **해석 보조 레이어 제공**
- 교육·연구·언론·기관에서 활용 가능한
    
    **표현 분석 데이터 인프라 구축**
    
- “뉴스를 판단하는 AI”가 아닌
    
    **“기준 기반 비교 도구”로 명확한 포지셔닝**
    

---

### Success Metrics (Revised)

- 표현 패턴 추출 정확도 ≥ **85%**
- 기준 적용 가능 기사 비율 ≥ **90%**
- 월 활성 사용자(MAU) ≥ **1,000명**
- 대시보드 평균 로딩 시간 ≤ **2초**
- 개인정보·저작권 이슈 **0건**
- 매체 비교 기능 재사용률 ≥ **30%**

---

## 3. 타겟 사용자 (Persona)

> BiasLens는 특정 이념이나 입장을 위한 서비스가 아니라,
> 
> 
> **뉴스를 ‘내 기준으로 해석하고 싶은 사용자’ 전반**을 대상으로 한다.
> 

---

### Persona 1 – 일반 뉴스 소비자 (Primary)

- 뉴스를 보지만 점점 피로감을 느끼는 사용자
- “왜 이렇게 느껴지는지”를 설명하기 어려움
- BiasLens를 통해:
    - 표현 차이를 빠르게 확인
    - 여러 매체를 기준에 따라 비교

---

### Persona 2 – 학술·교육 사용자

- 언론·사회·정치·미디어 전공자
- 표현 방식 분석을 정량적으로 설명하고 싶음
- BiasLens를 통해:
    - 사례가 아닌 **데이터 기반 설명 자료 확보**

---

### Persona 3 – 언론 및 콘텐츠 종사자

- 기자·에디터·콘텐츠 기획자
- 자사·타사 보도의 표현 경향을 점검하고 싶음
- BiasLens를 통해:
    - 시장 내 상대적 표현 위치 인식

---

### Persona 4 – 기업·기관 실무자

- 홍보·정책·공공 커뮤니케이션 담당
- 특정 이슈에 대한 언론 표현 경향 모니터링 필요
- BiasLens를 통해:
    - 수작업 기사 정리 비용 절감
    - 기준 기반 리포트 활용

---

## 4. User Story (Revised)

### US-1: 기준 설정 후 표현 차이 확인

```
AS A 뉴스 소비자
I WANT TO 내가 중요하게 생각하는 기준을 선택하고
SO THAT I CAN 매체별 표현 차이를 비교해 볼 수 있다

```

**Acceptance Criteria**

- 기준 선택 UI 제공
- 기준 변경 시 즉시 결과 반영
- 기준 요약 항상 노출

---

### US-2: 기사 단위 표현 분석

```
AS A 교육/연구 사용자
I WANT TO 특정 기사들의 표현 특징을 확인하여
SO THAT I CAN 분석 자료로 활용할 수 있다

```

---

### US-3: 매체 간 표현 비교

```
AS A 언론 종사자
I WANT TO 두 매체의 표현 경향을 비교하여
SO THAT I CAN 보도 스타일 차이를 설명할 수 있다

```

---

### US-4: 기간별 표현 변화 추이

```
AS A 기업/기관 사용자
I WANT TO 기간별 표현 변화 추이를 확인하여
SO THAT I CAN 이슈에 따른 보도 톤 변화를 파악할 수 있다

```

---

## 5. 비목표 (Non-Goals)

- ❌ 정치적 옳고 그름 판단
- ❌ 편향 여부 단정
- ❌ 개인 맞춤 뉴스 추천
- ❌ SNS·댓글·여론 분석
- ❌ 해외 뉴스 분석
- ❌ 모바일 네이티브 앱 (웹 중심)

---

## 6. 기능 요구사항 (Functional Requirements)

### FR-1 기준 설정

- 표현 기준 선택 (다중)
- 기준 설명 제공
- 기준 변경 기록 유지

---

### FR-2 대시보드

- 기준 요약 카드
- 매체별 표현 차이 지표
- 기간 선택 (7일 / 30일)
- 데이터 충분도 표시

---

### FR-3 기사 탐색

- 키워드 검색
- 날짜 / 매체 / 주제 / 기준 필터
- 표현 특징 태그 제공

---

### FR-4 매체 비교

- 2개 매체 선택
- 기준별 표현 분포 비교
- 차이 수치 제공
- 해석 가이드 문구 고정

---

### FR-5 리포트 생성 (Reference)

- 기준 기반 PDF 리포트
- 다운로드 / 공유
- 보관 기간 제한 (30일)
- 참고용 면책 문구 자동 삽입

---

## 7. 기술 요구사항 (Technical Requirements)

### AI / Data

- 표현 패턴 추출 모델
    - 감정어
    - 가치 판단 어휘
    - 단정적 서술
    - 방향성 표현
- 사용자 기준 매핑 로직
- 매체 단위 Aggregation

---

### 성능 / 보안

- API 응답 < 500ms (95%)
- 초기 로딩 < 2초
- 개인정보 자동 마스킹
- HTTPS / AES-256 암호화

---

## 8. Edge Case & Risks

### Edge Case

- 기준 적용 어려운 기사
- 기사 수 부족
- 중복 기사

### 대응

- 기준 적용 불가 표시
- 데이터 부족 안내
- URL 기반 중복 제거

---

## 9. Acceptance Criteria (Revised)

### 기능 / 품질

- 기준 적용 성공률 ≥ 90%
- 지표 오차 ±1%
- 치명적 버그 0건

### 보안 / 윤리

- 개인정보 노출 0건
- AI 판단·단정 표현 0건
- 기준 기반 설명 100% 적용

---

## 📌 최종 정리 문장 (PRD용)

> BiasLens는 편향을 판단하지 않는다.사용자가 선택한 기준에 따라뉴스 표현의 차이를 계산해 보여줄 뿐이다.
>
